# App Store Screenshots

## Architecture

Screenshots are generated by a three-stage pipeline:

1. **Raw capture** — `scripts/capture-appstore-screenshots.ts` (Playwright)
   - Launches Chromium, activates demo mode, navigates to trips, clicks tabs
   - Outputs raw PNGs to `iPhone-6.7/` and `iPad-Pro-12.9/`

2. **Marketing composites** — `appstore/scripts/generate_marketing_screenshots.py` (Pillow)
   - Takes raw PNGs, adds device frame + headline text + branded background
   - Outputs to `marketing/iPhone-6.7/`

3. **Fastlane prep** — `appstore/scripts/prepare_fastlane_screenshots.py`
   - Copies marketing PNGs to `marketing/en-US/` for `fastlane deliver`

**Fastlane** handles upload only (`upload_screenshots`, `upload_marketing_screenshots` lanes), not generation.

## Deterministic Rerun Commands

From repo root:

```bash
# Build the app first (required for localhost capture)
npm run build

# Start local preview server (in another terminal)
npm run preview -- --port 4173

# iPhone 6.7" only
npm run screenshots:appstore:iphone

# iPad Pro 12.9" only
npm run screenshots:appstore:ipad

# Both device classes
npm run screenshots:appstore:all

# Rebuild marketing composites after raw captures
npm run screenshots:marketing

# Prep for Fastlane upload
python3 appstore/scripts/prepare_fastlane_screenshots.py
```

Fastlane wrapper lanes (from `ios-release/`):

```bash
bundle exec fastlane ios generate_store_screenshots_iphone
bundle exec fastlane ios generate_store_screenshots_ipad
bundle exec fastlane ios generate_store_screenshots_all
```

### Single-shot rerun

```bash
npm run screenshots:appstore -- --only=03-calendar-itinerary --device=iphone
```

## Readiness Gates

The capture script does NOT use fixed delays. For each tab screenshot it verifies:

1. Trip route is loaded and not on auth gate or "Not Found"
2. Tab button is clicked and `[data-tab][data-active="true"]` confirms activation
3. Skeleton/loading indicators (`.animate-pulse`, `.animate-spin`, `"Loading..."`) have cleared
4. Tab-specific content selectors confirm real UI is rendered (calendar grid, chat messages, payment cards, place cards, media grid, poll cards)
5. Output PNG dimensions match App Store requirements

A machine-readable report is written to:

```
appstore/screenshots/capture-report.json
```

If any screenshot fails all fallback trips, the script exits with code 1.

## Trip Diversity

Each screenshot uses a different demo trip to avoid repetitive App Store panels:

| Screenshot | iPhone Trip | iPad Trip |
|-----------|------------|-----------|
| 01 Home Dashboard | /demo | /demo |
| 02 Chat | /trip/1 | /trip/4 |
| 03 Calendar | /trip/2 | /trip/8 |
| 04 Concierge/Places | /trip/8 (concierge) | /trip/2 (places) |
| 05 Payments | /trip/4 | — |
| 06 Places | /trip/3 | — |
| 07 Media | /trip/5 | — |
| 08 Polls | /trip/6 | — |

To change trips, edit the `IPHONE_SHOTS` / `IPAD_SHOTS` arrays in `scripts/capture-appstore-screenshots.ts`. Each shot has `fallbackTrips` for automatic retry if the primary trip lacks data.
